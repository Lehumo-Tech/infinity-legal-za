Architecture & Technology

Technical

Clean Architecture (Strict TypeScript)

Business logic is fully separated from UI — enabling independent testing, easier maintenance, and team scalability.

File Structure — lib/modules/ (Pure TS, Zero Framework Imports)

lib/ ├── modules/ │ ├── intake/ ← Phase 1: PUBLIC WIZARD │ │ ├── schema.ts Zod validation (14 fields, SA phone regex) │ │ ├── workflow.ts Pure business logic (validate, build, conflict) │ │ └── index.ts Barrel exports │ ├── cases/ ← Phase 2: CASES MODULE │ │ ├── types.ts CaseRecord, CaseStatus, Urgency types │ │ ├── schema.ts Zod schemas (create, update, filters) │ │ ├── utils.ts Stats, filtering, sorting, status transitions │ │ └── index.ts Barrel exports │ └── [leads, docs...] ← Phase 3: Upcoming ├── intake/ ← Staff Intake Service (API client layer) │ ├── types.ts, schemas.ts, utils.ts, service.ts hooks/ ├── useIntakeWizard.ts Bridge: Pure logic → React Hook Form ├── useCases.ts Bridge: Cases module → React state ├── useIntakeList.ts Bridge: Staff intake list → React ├── useIntakeConvert.ts Bridge: Conversion workflow → React
